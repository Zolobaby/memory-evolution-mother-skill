# 🧬 Memory Evolution 母 Skill

> 把一次次“会做事”，升级为可验证、可复用、可持续进化的能力系统。

---

## 它是什么？

Memory Evolution 是一个给 AI Agent 使用的“进化中枢”。
它把日常执行过程里的经验，沉淀为可复用资产，并通过日志与效果回执持续优化。

不是多一个脚本，而是多一个“器官”。

---

## 它解决什么问题？

很多 Agent 都能完成任务，但会反复踩坑：

- 同类问题重复学习
- 经验散落在聊天和日志里
- 分享了能力，却无法证明效果
- 新后端/新容器接入时，迁移成本高

Memory Evolution 给出一条闭环：

**捕获（Capture）→ 串联（Chain）→ 验证（Verify）→ 升级（Upgrade）**

---

## 核心能力（4层）

### 1) 三层记忆联动（Triad）
- L1 本地锚点（快速、稳定）
- L2 语义记忆层（可选、可降级）
- L3 增量蒸馏层（长期高价值）

支持 `full / core-only / degraded` 运行模式，避免单点失效。

### 2) 反重复进化链（Anti-Repeat Chain）
- 事件按 track 串联
- fingerprint 去重
- 自动找到“最近可复用基线”

目标：不再每次从 0 开始。

### 3) 效果回执（Effect Receipt）
每次能力复用后记录：
- 是否成功
- 节省了多少时间
- 避免了多少重复

让“能力分享”从主观感觉变成可验证数据。

### 4) 容器接纳协议（Container Contract）
定义统一接口与迁移策略，未来替换更强记忆容器时可平滑切换。

---

## 为什么它像 Mini-EvoMap？

因为它已经具备小型能力市场的核心要素：

- 资产化：经验可打包
- 索引化：相近能力可串联
- 验证化：效果可量化
- 治理化：有边界、有审计

如果多人共享，就会形成一个高信任、快迭代的私域进化网络。

---

## 安全边界（非常重要）

- 不越权：只能使用安装者“已安装 + 已授权”能力
- 可审计：探针本地优先，可检测、可关闭
- 默认最小化：不采集密钥，不自动外发敏感信息
- 可降级：某层不可用时，不阻断核心链路

一句话：这是“体检 + 路由”机制，不是提权机制。

---

## 适用场景

- 个人 Agent 的长期能力进化
- 团队多 Agent 协作复盘与迁移
- 高重复工作流的稳定提效
- 从“自动化执行”走向“自动化进化”
